# 记忆系统

只有那些跨会话仍然有价值，而且不能轻易从当前仓库状态直接推出来的信息，才适合进入 memory。

## 名词解释

`跨会话`：当前会话结束，下一次重新开始一个新的会话；

`不可轻易推导`：不是扫一边代码可以轻易得到的知识，例如用户的偏好；

## 常见的四类memory

1、`user`：用户偏好

2、`feedback`：用户纠正过的地方

3、`project`：不轻易从代码中直接看出来的项目约定和背景

4、`reference`：外部资源指针

**哪些东西不要存进memory：**


| 不要存的东西 | 为什么 |
| :-----       | :--:   |
|文件结构、函数签名、目录布局|这些可以重新读代码得到|
|当前任务进度|这属于 task / plan，不属于 memory|
|临时分支名、当前 PR 号|很快会过时|
|修 bug 的具体代码细节|代码和提交记录才是准确信息|
|密钥、密码、凭证|安全风险|

## 最小心智模型

```text
conversation
   |
   | 用户提到一个长期重要信息
   v
save_memory
   |
   v
.memory/
  ├── MEMORY.md        # 索引
  ├── prefer_tabs.md
  ├── feedback_tests.md
  └── incident_board.md
   |
   v
下次新会话开始时重新加载
```

```python
def save_memory(name, description, type, content):
    path = memory_dir / f"{safe_name}.md"
    path.write_text(frontmatter + content)
    rebuild_index()
```
