# 错误恢复

把“报错就崩”升级成“先判断错误类型，再选择恢复路径”

## 名词解释

`恢复`：当出现问题时，先补救，如果补救失败，再把明确错误告知用户；

`重试预算`：最多重试几次，防止程序无限循环；

`状态机`：在几个明确的状态之间按规则切换；

## 最小心智模型

3类问题：

```
1. 输出被截断
   模型还没说完，但 token 用完了

2. 上下文太长
   请求装不进模型窗口了

3. 临时连接失败
   网络、超时、限流、服务抖动
```

对应3种恢复手段：

```
LLM call
  |
  +-- stop_reason == "max_tokens"
  |      -> 注入续写提示
  |      -> 再试一次
  |
  +-- prompt too long
  |      -> 压缩旧上下文
  |      -> 再试一次
  |
  +-- timeout / rate limit / transient API error
         -> 等一会儿
         -> 再试一次
```
